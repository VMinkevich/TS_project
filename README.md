# Volatility Regimes and Forecasting Performance

Проект по прогнозированию финансовых временных рядов с целью проверки гипотезы: “Режимы рынка влияют на качество модели"

Гипотезы:
* В моменты высокой волатильности, результаты прогноза хуже (больше RMSE/MAE, ниже Directional Accuracy)
* GARCH модель более стабильна по сравнению с другими моделями с точки зрения изменения метрик между режимами
* Точность прогноза любой модели сильно зависит от семейства активов (S&P500 vs EUR/USD)

---

## Данные

Дневные данные:

* S&P 500 index (`^GSPC`)
* EUR/USD exchange rate (`EURUSD=X`)

Период:

* Январь 2023 — Декабрь 2025

Источник данных:

* Yahoo Finance (via `yfinance`)

Разбиение:
Трейн: 80% данных (до 2025-01-01)
Тест: 20% данных (после 2025-02-01)

Разбиение на high/low volatility:
* Вычисляем скользящую волатильность (окно = 20 дней)
* Определяем режимы:
  * Высокая волатильность: выше медианы
  * Низкая волатильность: ниже медианы

---

## Модели

Мы сравниваем модели из разных семейств:

* Statistical:
  * Naive (последнее наблюдение)
  * GARCH-семейство (GJR-GARCH) для оценки волатильности

* Machine Learning:
  * Gradient Boosting (XGBoost)

* Deep Learning:
  * Neural Network (MLP из sklearn, CPU-версия) для простоты и доступности

---

## Методология

1. Считаем лог-доходности для обоих активов
2. Оцениваем скользящую волатильность (окно = 20)
3. Определяем режимы волатильности:
   * Высокая волатильность: выше медианы
   * Низкая волатильность: ниже медианы
4. Обучаем модели на train
5. Делаем прогнозы на test
6. Считаем метрики по каждому режиму
7. Сравниваем результаты между режимами и моделями

---

## Метрики

* RMSE
* MAE
* Directional Accuracy
* Delta = Error_high_vol − Error_low_vol

## Пайплайн делает
Pipeline делает:
Удаление старых результатов, загрузку и предобработку данных, обучение моделей, прогнозирование и сохранение метрик по режимам в JSON.

Данные:
- Загрузку и сохранение данных S&P500 и EUR/USD
- Предобработку и вычисление returns
- Разделение на train/test
- Выделение high_vol / low_vol на тесте
Обучение моделей:
- Naive последнего наблюдения
- GJR-GARCH
- XGBoost с 10 признаками
- MLP Regressor на CPU
Процедура валидации и тюнинга гиперпараметров:
- Пропустим для простоты
Тест:
- Прогноз на тестовом периоде
- Сохранение метрик по regime в JSON (results/tables/metrics.json)
